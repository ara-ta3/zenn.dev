# Output Format

ユーザーが別形式を求めていない限り、記事の方向性は次の構成で出す。

## 1. 結論

- 次に書くべき記事の方向性を 2-4 文で述べる。
- 読者、扱う問題、既存記事との差分を必ず含める。

## 2. 根拠になった既存記事

- 3 本以内に絞る。
- 各記事について次を短く書く。
  - ファイルパス
  - 何を既にカバーしているか
  - 今回の提案にどうつながるか

## 3. 推奨する方向性

- 想定読者
- 解決したい具体的な悩み
- 新しく提供する価値
- 既存記事との差別化ポイント

## 4. タイトル案

- 3 案出す。
- 少なくとも 1 案は「何が増えるのか」が見える具体的なタイトルにする。

## 5. 構成案

- H2 ベースで 4-6 項目程度にする。
- 既存記事と重複する章は作らず、必要なら「前回記事のおさらいは最小限」と明記する。

## 6. 書かないこと

- 今回の方向性では扱わない内容を書く。
- スコープが広がりやすい場合ほど明示する。

## 7. 確認したいこと

- 不確定要素がある場合だけ書く。
- ユーザーに確認すべき前提、環境、対象読者のズレを書く。
